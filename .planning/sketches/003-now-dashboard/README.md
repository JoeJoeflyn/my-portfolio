# Sketch 003: Now Dashboard

## Design Question
How should the WakaTime + GitHub heatmaps and stats compose into a single
unified "currently / now" section?

## How to View
open .planning/sketches/003-now-dashboard/index.html

## Variants
- **A: Stacked Terminal Report** — Vertical stack: status bar, stat row, then two heatmaps full-width. Reads like a `git status` + activity report.
- **B: Two-Column Activity** — Left: GitHub heatmap + commit stats. Right: WakaTime heatmap + language breakdown. Side-by-side comparison.
- **C: Command-Center Bento** — Asymmetric grid: big heatmap anchor, stat tiles, language bars, "currently building" card. Dense dashboard.

## What to Look For
- Does stacking read like a report, or feel like a long scroll with no hierarchy?
- Does two-column comparison help, or split focus awkwardly on mobile?
- Does the bento command-center surface too much, or just right?
- Which makes the heatmaps &mdash; the strongest visual asset &mdash; the hero?
