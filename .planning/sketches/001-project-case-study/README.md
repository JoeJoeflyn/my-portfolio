# Sketch 001: Project Case Study

## Design Question
How should a project detail/case-study page feel? Projects currently link externally;
this explores an in-site case-study view (problem -> approach -> outcome -> stack).

## How to View
open .planning/sketches/001-project-case-study/index.html

## Variants
- **A: IDE Split** — File-tree sidebar nav + main content pane. Feels like opening a project in an editor.
- **B: Linear Long-Read** — Full-width stacked sections, big numbered headers, hard-shadow cards. Brutalist narrative.
- **C: Bento Grid** — Asymmetric modular cards, each a facet (problem, stack, metrics, gallery). Dashboard feel.

## What to Look For
- Does the IDE metaphor reinforce the "developer workstation" feel, or feel gimmicky?
- Does linear scroll read like a story, or feel like a wall of text?
- Does bento grid surface the right info at a glance, or fragment the narrative?
- How does each handle the outcome/metrics block — the part that proves the work?
