# Sketch 002: Hero Rethink

## Design Question
Does the current big-name + terminal.txt hero land, or is there a stronger intro
that still fits the brutalist/terminal system?

## How to View
open .planning/sketches/002-hero-rethink/index.html

## Variants
- **A: Current Baseline** — Reproduces the existing hero (big name + terminal.txt box) as the reference point.
- **B: Interactive Terminal** — A working fake terminal that types the intro on load and accepts commands (whoami, ls, help). Reinforces "developer workstation."
- **C: Split + Now Panel** — Left: name + role + CTAs. Right: a live "now" panel (local time, status, current focus). More structured, less monolithic.

## What to Look For
- Does the interactive terminal earn its complexity, or feel like a gimmick?
- Does the split layout surface useful info, or dilute the name impact?
- Is the current baseline still the strongest, or does it feel flat next to the alternatives?
- How does each handle the first 3 seconds &mdash; what does a visitor actually do?
